20 /*
21  * Array is a class that inherits from Kernel_Array and the Allocator type.
22  * The Array class is used to initialize and destruct the Kernel_Array object.
23  *
24  * Array and Kernel_Array are two tightly coupled classes as expression-templates must be trivially copyable (to pass them to CUDA functions).
25  * Separating these two enables the usage of non-trivially copyable allocators as well as the ability to define
26  * non-default move and copy assignments/constructors.
27  *
28  * The Kernel_Array class should never be instantiated normally. It should only be accessed by instantiating
29  * an instance of the Array class, and calling 'my_array_object.internal()' to query it.
30  *
31  * Additionally this design pattern (replicated in Array_View) enables expression-templates to
32  * defines additional members that we do not want to pass to the GPU. (As they my be non-essential to the computation).
33  *
34  */
